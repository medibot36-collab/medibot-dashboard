HELP & DOCUMENTATION

🔌 Firebase Schema

  • Vitals path: vitals/{patientId}/hr, spo2, timestamp
  • Patient info: patients/{patientId}/name, age, gender, email, createdAt
  • PDF reports: reports/{patientId}/pdfUrl
  • Active patient: Commands/activePatient

📄 PDF Report Flow

  • Backend generates a PDF and uploads to Firebase Storage.
  • Backend writes the download URL to reports/{patientId}/pdfUrl.
  • Dashboard listener fires immediately — no refresh needed.
  • Click View Report to always open the latest PDF.

❤️ Sensor Readings

  • MAX30102 provides HR and SpO₂ data.
  • ESP32 writes to vitals/{patientId}/ in Firebase.
  • Z-score > 2 triggers Warning; threshold breach → Danger.
  • Risk meter turns red on critical readings automatically.

🤖 Robot Control

  • Drag the joystick or use arrow keys to move.
  • Commands write to Commands/robot/ in real-time.
  • Active patient (set in Firebase) shows in the robot panel.
  • Always press STOP before releasing the joystick.
